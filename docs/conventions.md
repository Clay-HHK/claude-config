# Naming Conventions & Standards

## Skill Naming
- Format: kebab-case (lowercase + hyphens)
- Form: prefer gerund form (verb+ing)
- Example: `scientific-writing`, `git-workflow`, `bug-detective`

## Tags Naming
- Format: Title Case
- Abbreviations all caps: TDD, RLHF, NeurIPS, ICLR
- Example: `[Writing, Research, Academic]`

## Description Standards
- Person: third person
- Content: include purpose and use cases
- Example: "Provides guidance for academic paper writing, covering top-venue submission requirements"

## Writing Style
- No AI-sounding writing patterns: avoid em-dashes, overly polished/robotic tone
- Banned words/phrases: 'delve', 'crucial', 'landscape', 'notably', 'leveraging', 'comprehensive', 'furthermore', 'utilize'
- Write in a natural, human academic tone with logical flow
- Social media copy should be conversational and authentic, not corporate
- LaTeX: no em-dashes, `\cite` commands must have no spaces after commas

## Figure and Visualization Style
- Keep figures clean and minimal, publication-ready by default
- No decorative elements: no arrows, crowded labels, or unnecessary legends
- Simple color schemes (max 4 colors), clear axis labels, readable font sizes
- Default to matplotlib with minimal customization unless specified otherwise
